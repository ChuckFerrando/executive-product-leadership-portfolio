## 3. Non-Negotiable PLOS Architecture Rules

### Core ownership
- **Strategy** → direction and refinement
- **Governance** → prioritization, tradeoffs, decisions
- **Delivery** → execution
- **Customer Outcomes** → interpretation, evaluation, value qualification, learning
- **Decision Intelligence** → signals, metrics, dashboards, descriptive analytics only
- **Playbooks** → execution patterns only
- **Labs** → controlled experimentation only

### Absolute rule
> **Never collapse evidence, meaning, and decision-making into the same layer.**

### No shadow systems
No shadow systems may be created.

- No implicit systems
- No renamed equivalents of existing systems
- No “blended” layers that combine responsibilities across systems

- **Evidence** = Decision Intelligence
- **Meaning** = Customer Outcomes
- **Decisions** = Strategy / Governance

### Mandatory mediation path
> **Decision Intelligence → Customer Outcomes → Strategy / Governance**

Never allow:
- DI → Strategy
- DI → Governance

Strategy must not refine itself from raw dashboards.
Governance must not act directly on raw signals.

### Playbooks
Must define **how work is executed**, not what the system means or decides.

Must NOT:
- make decisions
- interpret DI signals
- evaluate outcomes
- bypass Outcomes or Governance
- create new system pathways
- contain experimentation as a canonical function

### Labs
Exist for **controlled experimentation only**.

All experiments must demonstrate a reusable capability, not just a one-off output.

Must NOT:
- act as production authority
- interpret evidence as canonical meaning
- bypass Outcomes
- bypass Governance
- become canonical by informal adoption

Promotion path must remain enforced:

> **Labs → Outcomes → Governance**

No experiment may be promoted directly into production systems without passing through Customer Outcomes validation and Governance decision.
