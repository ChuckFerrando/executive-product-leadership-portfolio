## Portfolio Logic

The repositories connect in the following way:

### Signal
External information, sources, and ideas are curated through the Signal Center.

### Thinking
Those signals are interpreted using the frameworks in Mental Models.

### Tools
Relevant capabilities and technologies are understood through AI Fluency.

### Build
Ideas are translated into experiments and prototypes in the Experimentation Lab.

### System Architecture
PLOS anchors the entire portfolio with the underlying product operating architecture.

## Standard Files Across Repositories

Each repository in this ecosystem follows a common structure:

- `README.md`
- `START_HERE.md`
- `REPO_MAP.md`

This is intentional and supports consistent navigation across the portfolio.

---

## Design Principle

The portfolio is designed as a coherent system of repositories.

Each repository has:
- a clear role
- a defined scope
- a primary purpose
- explicit relationships to the others

This structure is intended to make the portfolio easy to understand, evaluate, and navigate.
